联动改校验规则

            
                
                {
                    ...组件的其它属性,
                     "target":"loanAppInfo.appTime1",
                    "definitionMap" : {
                        "valueA1" : {
                            "rules" : [
                                {
                                    "required" : true,
                                    "pattern" : "/(^1[0-9][0-9][0-9](.[0-9]{1,2})?$)|(^1[0-9][0-9][0-9]$)/",
                                    "message" : "必填, 金额输入范围为1000~2000,最多两位小数"
                                }
                            ]
                        },
                        "valueA2" : {
                            "rules" : [
                                {
                                    "required" : true,
                                    "message" : "必填, 金额输入范围为2000~3000,最多两位小数",
                                    "pattern" : "/(^2[0-9][0-9][0-9](.[0-9]{1,2})?$)|(^2[0-9][0-9][0-9]$)/"
                                }
                            ]
                        }
                    },
                }

注意本配置是配在被联动组件上的，例如A的值变化，影响到B的校验规则，则在B组件上配置本信息

目前只有InputNumebr支持该属性，后续新增支持组件会在此说明

target: 在上面的例子里，就是A的path

definitionMap: A的值和B的校验规则之间的映射关系

valueA: A组件的值
